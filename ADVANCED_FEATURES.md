# Расширенные возможности disk_cleanup.sh

## Дата обновления: 24 февраля 2026 г.

## Добавленные опции командной строки

### 1. Управление размером журналов (`-j, --max-journal-size`)

**Описание:** Устанавливает максимальный размер журналов systemd  
**Формат:** Число + единица (K, M, G)  
**Дефолт:** 500M  
**Валидация:** Проверка формата, fallback на дефолт при ошибке

**Примеры:**
```bash
--max-journal-size 1G      # 1 гигабайт
--max-journal-size 200M    # 200 мегабайт
-j 2G                      # короткая форма
```

### 2. Возраст кэша миниатюр (`-t, --thumbnail-age`)

**Описание:** Количество дней для хранения миниатюр  
**Формат:** Целое число (дни)  
**Дефолт:** 30  
**Валидация:** Минимум 1 день

**Примеры:**
```bash
--thumbnail-age 60         # хранить 60 дней
-t 90                      # хранить 90 дней
```

### 3. Количество строк при обрезке (`-l, --truncate-lines`)

**Описание:** Сколько строк оставлять в больших лог-файлах  
**Формат:** Целое число  
**Дефолт:** 10000  
**Валидация:** Минимум 100 строк

**Примеры:**
```bash
--truncate-lines 5000      # оставить 5000 строк
-l 20000                   # оставить 20000 строк
```

### 4. Минимальный размер для обрезки (`-s, --min-log-size`)

**Описание:** Минимальный размер лог-файла в MB для обрезки  
**Формат:** Целое число (мегабайты)  
**Дефолт:** 100  
**Валидация:** Минимум 1 MB

**Примеры:**
```bash
--min-log-size 50          # обрезать файлы >50MB
-s 200                     # обрезать файлы >200MB
```

### 5. Пропуск задач (`-S / --skip`)

**Описание:** Пропустить определенные задачи очистки  
**Формат:** Список через запятую (без пробелов)  
**Дефолт:** Выполнять все  
**Валидация:** Нет (неизвестные задачи игнорируются)

**Доступные задачи:**
- `journals` - systemd journal cleanup
- `oldlogs` - compressed/rotated logs cleanup
- `apt` - APT cache cleanup
- `kernels` - old kernel removal
- `packages` - unused packages removal
- `snap-revisions` - old snap revisions removal
- `snap-cache` - snap cache cleanup
- `thumbnails` - thumbnail cache cleanup
- `temp` - temporary files cleanup
- `truncate` - large log files truncation

**Примеры:**
```bash
-S apt                         # пропустить только APT
--skip apt,kernels             # пропустить APT и ядра
-S snap-revisions,snap-cache,thumbnails  # пропустить несколько snap задач
```

---

### 6. Выполнение только указанных задач (`-T / --tasks`)

**Описание:** Выполнить ТОЛЬКО указанные задачи  
**Формат:** Список через запятую (без пробелов)  
**Дефолт:** Выполнять все  
**Комбинация:** Можно использовать вместе с `-S` для исключения некоторых задач

**Доступные задачи:** те же, что и для `-S` (см. выше)

**Примеры:**
```bash
-T journals                        # выполнить только очистку журналов
--tasks journals,oldlogs           # только журналы и старые логи
-T apt,packages,snap-cache         # только apt, пакеты и snap-кэш
```

**Когда использовать `-T` вместо `-S`:**
- Когда нужно выполнить 2-3 задачи из 10 → `-T` короче
- Когда нужно быстро почистить что-то конкретное → `-T journals`
- Когда хотите исключить 8 задач → лучше `-T task1,task2`

**Когда использовать `-S` вместо `-T`:**
- Когда нужно пропустить 1-2 задачи → `-S kernels`
- Когда нужно выполнить почти все → проще указать что пропустить

---

### 7. Комбинирование `-T` и `-S`

🔧 **Можно использовать обе опции одновременно!**

**Логика работы:**
1. Сначала применяется `-T` (фильтр только нужных задач)
2. Затем применяется `-S` (исключение из оставшегося списка)

**Примеры:**
```bash
# Выполнить journals и oldlogs (исключить apt)
-T journals,oldlogs,apt -S apt
# Результат: выполнятся только journals, oldlogs

# Выполнить только journals
-T journals,apt,temp -S apt,temp
# Результат: выполняется только journals

# Все snap-задачи кроме snap-cache
-T snap-revisions,snap-cache -S snap-cache
# Результат: выполняется только snap-revisions
```

## Улучшенный --dry-run

Режим превью теперь показывает:
- ✅ Все текущие настройки (параметры)
- ✅ Список задач, которые будут выполнены
- ✅ Указание `-T` (если используется)
- ✅ Указание `-S` (если используется)
- ✅ Визуальное отображение (✓ для каждой задачи)

**Пример вывода:**
```

========================================

Configuration:
  Days to keep logs: 14
  Max journal size: 1G
  Thumbnail max age: 60 days
  Log truncate lines: 5000

  Skipping tasks: apt,kernels

Tasks that would be executed:
  ✓ Clean systemd journals
  ✓ Remove old log files
  ✓ Remove unused packages
  ✓ Clean snap revisions
  ✓ Clean snap cache
  ✓ Clean thumbnail cache
  ✓ Clean temporary files
  ✓ Truncate large log files

========================================
```

## Валидация параметров

Все параметры проходят валидацию:

| Параметр | Валидация | При ошибке |
|----------|-----------|-----------|
| `--days` | Целое число ≥ 1 | Дефолт: 7, предупреждение |
| `--max-journal-size` | Формат: число+K/M/G | Дефолт: 500M, предупреждение |
| `--thumbnail-age` | Целое число ≥ 1 | Дефолт: 30, предупреждение |
| `--truncate-lines` | Целое число ≥ 100 | Дефолт: 10000, предупреждение |
| `--min-log-size` | Целое число ≥ 1 | Дефолт: 100, предупреждение |
| `-S/--skip` | Список строк | Неизвестные задачи игнорируются |
| `-T/--tasks` | Список строк | Неизвестные задачи игнорируются, комбинируется с `-S` |

**Пример валидации:**
```bash
$ disk_cleanup.sh -d abc --max-journal-size invalid --dry-run
Error: Invalid number of days: abc
Using default (7 days).
Warning: Invalid journal size format: invalid
Using default (500M). Format should be like: 100M, 1G, etc.
# ... затем продолжает с дефолтными значениями
```

## Комбинированные примеры использования

### Сценарий 1: Рабочая станция (часто включается/выключается)
```bash
# Консервативная очистка, не трогать ядра
disk_cleanup.sh -d 30 --skip kernels
```

### Сценарий 2: Сервер с ограниченным местом
```bash
# Агрессивная очистка
disk_cleanup.sh -d 3 -s 50 -l 5000 -j 200M
```

### Сценарий 3: Сервер разработки
```bash
# Средняя очистка, пропустить долгие операции
disk_cleanup.sh -d 14 --skip kernels,packages
```

### Сценарий 4: Производственный сервер
```bash
# Безопасная очистка с большими буферами
disk_cleanup.sh -d 30 -j 2G -s 200 --skip kernels
```

### Сценарий 5: Тестирование новых настроек
```bash
# Сначала dry-run
disk_cleanup.sh -d 7 -s 30 --dry-run
# Если все ок, запустить
disk_cleanup.sh -d 7 -s 30
```

### Сценарий 6: Быстрая очистка конкретных задач
```bash
# Очистить только журналы и apt-кэш (быстро без удаления пакетов/ядер)
disk_cleanup.sh -T journals,apt

# Очистить только snap-связанные задачи
disk_cleanup.sh --tasks snap-revisions,snap-cache
```

### Сценарий 7: Комбинирование -T и -S для точного контроля
```bash
# Выполнить journals и oldlogs (исключить apt из списка)
disk_cleanup.sh -T journals,oldlogs,apt -S apt

# Выполнить несколько задач, но исключить некоторые (например, для тестирования)
disk_cleanup.sh -T journals,apt,temp,packages -S packages

# Все кроме одной задачи (например, все snap кроме cache)
disk_cleanup.sh -T snap-revisions,snap-cache -S snap-cache
```

## Внутренние изменения

### Новая утилитарная функция

```bash
should_skip_task() {
    local task="$1"
    if [[ ",$SKIP_TASKS," == *",$task,"* ]]; then
        return 0  # true - skip this task
    fi
    return 1  # false - don't skip
}
```

### Использование в main()

```bash
# Вместо прямого вызова:
clean_systemd_journals

# Теперь с проверкой:
should_skip_task "journals" || clean_systemd_journals
```
